基于hexo的个人主页创建

个人主页创建

原理

  1. 博客生成框架- hexo

    把输入：YML（进行博客设置），Markdown（博客内容书写） 直接生成包含三种文件的网站目录:public，其中，三种网站目录是：HTML，CSS，JS。

    其中HTML用来设置网站样式，CSS用来设置网站样式，JS用来描述网站上按钮等行为

  2. 文件托管平台

    就是在github上存放网站目录

  3. 站点部署服务

    把已经成型的网站部署到互联网上：

    1. 使用将public部署到git上的插件，使通过yml配置部署地点直接将hexo部署到yml中制定的位置。
    2. 更改yml。

仓库创建+github绑定

域名绑定（目前已省略）

ssh配置

使用hexo搭建个人博客

将个人博客部署到github上 ==这一段不是很理解==

  1. 安装插件 hexo-deployer-git

    其中，插件的作用是：将生成的静态网站文件部署到一个远程Git仓库中，以便将您的网站内容发布到互联网上。

    具体来说，hexo-deployer-git插件的作用包括以下几个方面：

    1. 将静态网站生成的HTML、CSS、JavaScript文件以及其他资源文件提交到指定的Git仓库。
    2. 自动创建或更新Git仓库中的分支，通常是gh-pages分支，以便托管您的网站内容。
    3. 提供配置选项，允许您指定Git仓库的URL、分支名称、提交消息等参数。
    4. 允许您使用命令行来轻松地执行部署操作，以便快速发布您的网站。

    通过使用hexo-deployer-git插件，您可以将Hexo生成的静态网站部署到像GitHub Pages、GitLab Pages等支持静态网站托管的平台，使您的网站能够在线访问。这个插件简化了网站发布过程，使您无需手动复制文件或配置远程服务器，只需运行一个命令即可完成部署操作。

  2. 配置_config.yml文件

    _config.yml是配置文件，用于配置和自定义您的Hexo网站。这个文件位于Hexo网站的根目录下，它包含了各种设置选项，以便您可以根据您的需求来配置网站的外观、功能和行为。_config.yml是一个YAML格式的文件，您可以使用文本编辑器打开它并编辑其中的内容。

    注意这里要改成main分支，估计是因为个人主页默认显示main分支，如果提交到master上就是空白页

更改个人博客主题

  1. 在官网上找主题，进入主题的github当中

  2. git clone https://github.com/ikeq/hexo-theme-inside.git themes/主题名 clone主题仓库到hexo文件当中==如何提高速度？== 注意主题名一定要对，并不是inside就是主题，不对应上会报错

  3. 配置_config.yml ：把主题设置成已下载的主题名

  4. hexo g重新生成

  5. hexo d上传
